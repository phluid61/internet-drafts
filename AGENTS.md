# Agents

This repository contains IETF Internet-Drafts authored in kramdown-rfc2629
markdown. It uses [martinthomson/i-d-template](https://github.com/martinthomson/i-d-template)
as its build system.

## Repository layout

- Active drafts are `draft-*.md` files in the repository root (e.g.
  `draft-kerwin-http2-encoded-data.md`, `draft-kerwin-rfc8089-bis-core.md`).
- `Published/` contains archived drafts that have become RFCs (e.g.
  `draft-ietf-appsawg-file-scheme.md`, which became RFC 8089).
- `lib/` is a shallow clone of
  [martinthomson/i-d-template](https://github.com/martinthomson/i-d-template),
  auto-cloned by the root `Makefile`. It is not committed to the repository.
- `.github/workflows/` contains CI workflows generated by i-d-template.
- `.github/` also contains issue and PR templates, and contributing
  guidelines.

### Legacy subdirectories

Some draft subdirectories (e.g. `http2-encoded-data/`, `file-scheme/`)
still exist and contain historical versioned `.xml` snapshots. These are
retained for reference but are not used by the build system. The
authoritative source for each draft is the root-level `.md` file.

## Source of truth

Each draft's authoritative source is its `draft-*.md` file in the
repository root. These are kramdown-rfc2629 documents (YAML front matter +
markdown body). The `docname` field in each draft uses the `-latest` suffix;
actual version numbers are derived from git tags.

PRs must update the relevant `draft-*.md` file(s). See
`.github/CONTRIBUTING.md` and `.github/PULL_REQUEST_TEMPLATE.md` for
contribution expectations, including the EDITORIAL / SUBSTANTIVE / TECHNICAL
classification.

## Build

The build system is [martinthomson/i-d-template](https://github.com/martinthomson/i-d-template).
The toolchain is `markdown ‚Üí xml ‚Üí txt + html`.

Prerequisites (not vendored; must be installed separately):

- [kramdown-rfc2629](https://github.com/cabo/kramdown-rfc2629)
- [xml2rfc](https://xml2rfc.ietf.org/)
- Python 3.10+ with `python3-venv`

Key make targets (run from the repository root):

- `make` ‚Äî build HTML and plain-text output for all drafts
- `make clean` ‚Äî remove generated files
- `make lint` ‚Äî run lint checks (whitespace, YAML, etc.)
- `make fix-lint` ‚Äî automatically fix lint errors where possible
- `make idnits` ‚Äî run nit checks (requires
  [idnits](https://tools.ietf.org/tools/idnits/))
- `make gh-pages` ‚Äî generate the gh-pages branch content
- `make next` ‚Äî bump to the next version number

Generated `.html`, `.txt`, and `.xml` files are git-ignored.

### CI workflows

- `ghpages.yml` ‚Äî builds drafts and deploys the editor's copy to gh-pages
- `publish.yml` ‚Äî submits drafts to the IETF datatracker on tag push
- `archive.yml` ‚Äî archives issues on draft submission
- `update.yml` ‚Äî updates i-d-template files
